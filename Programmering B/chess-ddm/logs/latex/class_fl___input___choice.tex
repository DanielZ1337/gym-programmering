\doxysubsection{Detailed Description}
A combination of the input widget and a menu button.

 
\begin{DoxyImage}
\includegraphics[width=6cm]{input_choice.jpg}
\doxyfigcaption{Fl\+\_\+\+Input\+\_\+\+Choice widget}
\end{DoxyImage}


The user can either type into the input area, or use the menu button chooser on the right to choose an item which loads the input area with the selected text.

The application can directly access both the internal \mbox{\hyperlink{class_fl___input}{Fl\+\_\+\+Input}} and \mbox{\hyperlink{class_fl___menu___button}{Fl\+\_\+\+Menu\+\_\+\+Button}} widgets respectively using the \mbox{\hyperlink{class_fl___input___choice_a7e863b5c9a096f52a9502619d11a7cd0}{input()}} and \mbox{\hyperlink{class_fl___input___choice_a7781575229c339905dd1e42a2451141f}{menubutton()}} accessor methods.

The default behavior is to invoke the \mbox{\hyperlink{class_fl___widget_ac32acdcfb3b722fccd2d3f946b01e799}{Fl\+\_\+\+Input\+\_\+\+Choice\+::callback()}} if the user changes the input field\textquotesingle{}s contents, either by typing, pasting, or clicking a different item in the choice menu.

The callback can determine if an item was picked vs. typing into the input field by checking the value of \mbox{\hyperlink{class_fl___input___choice_a7781575229c339905dd1e42a2451141f}{menubutton()}}-\/$>$\mbox{\hyperlink{class_fl___input___choice_aba5d22d415138b603a4913bfe46e90f0}{changed()}}, which will be\+: \begin{DoxyVerb}- 1: the user picked a different item in the choice menu
- 0: the user typed or pasted directly into the input field
\end{DoxyVerb}

Changes the size or position of the widget.

This is a virtual function so that the widget may implement its own handling of resizing. The default version does {\itshape not} call the \mbox{\hyperlink{class_fl___widget_aa63ce68cbf4620cf8750b868368ea02b}{redraw()}} method, but instead relies on the parent widget to do so because the parent may know a faster way to update the display, such as scrolling from the old position. ~\newline


Some window managers under X11 call \mbox{\hyperlink{class_fl___input___choice_a03c0a02d26ef3498a67d4868be32b552}{resize()}} a lot more often than needed. Please verify that the position or size of a widget did actually change before doing any extensive calculations.

position(\+X, Y) is a shortcut for resize(X, Y, \mbox{\hyperlink{class_fl___widget_ad59ece778aaf15acc4dd01a8bf5f8012}{w()}}, \mbox{\hyperlink{class_fl___widget_aeeabd61e2d10c76cd6600dbbfe951f71}{h()}}), and size(\+W, H) is a shortcut for resize(\mbox{\hyperlink{class_fl___widget_abea0b9ad493d58bf7fda8d3c81d477b7}{x()}}, \mbox{\hyperlink{class_fl___widget_aec1aded9efdf5f23dfac8cc725cefc4a}{y()}}, W, H).
