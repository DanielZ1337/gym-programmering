\doxysubsection{Detailed Description}
The \mbox{\hyperlink{class_fl___double___window}{Fl\+\_\+\+Double\+\_\+\+Window}} provides a double-\/buffered window. If possible this will use the X double buffering extension (Xdbe). If not, it will draw the window data into an off-\/screen pixmap, and then copy it to the on-\/screen window. 

It is highly recommended that you put the following code before the first \mbox{\hyperlink{class_fl___double___window_a4d1c3cbb1184c6785df7465b0c657025}{show()}} of {\itshape any} window in your program\+: 


\begin{DoxyCode}{0}

{\footnotesize\ttfamily \mbox{\hyperlink{png_8h_aa8c59027f9ab2769342f248709d68d17}{void}} Fl\+\_\+\+Double\+\_\+\+Window\+::resize (\begin{DoxyParamCaption}\item[{int}]{X,  }\item[{int}]{Y,  }\item[{int}]{W,  }\item[{int}]{H }\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [virtual]}}

Changes the size and position of the window. If \mbox{\hyperlink{class_fl___window_aa09300542b2d8b49fc69a037bdd3b742}{shown()}} is true, these changes are communicated to the window server (which may refuse that size and cause a further resize). If \mbox{\hyperlink{class_fl___window_aa09300542b2d8b49fc69a037bdd3b742}{shown()}} is false, the size and position are used when \mbox{\hyperlink{class_fl___double___window_a4d1c3cbb1184c6785df7465b0c657025}{show()}} is called. See \mbox{\hyperlink{class_fl___group}{Fl\+\_\+\+Group}} for the effect of resizing on the child widgets.

You can also call the \mbox{\hyperlink{class_fl___widget}{Fl\+\_\+\+Widget}} methods size(x,y) and position(w,h), which are inline wrappers for this virtual function.

A top-\/level window can not force, but merely suggest a position and size to the operating system. The window manager may not be willing or able to display a window at the desired position or with the given dimensions. It is up to the application developer to verify window parameters after the resize request.

\doxysubsubsection{\texorpdfstring{show()}{show()}\hspace{0.1cm}{\footnotesize\ttfamily [1/2]}}
{\footnotesize\ttfamily \mbox{\hyperlink{png_8h_aa8c59027f9ab2769342f248709d68d17}{void}} Fl\+\_\+\+Double\+\_\+\+Window\+::show (\begin{DoxyParamCaption}{ }\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [virtual]}}

Puts the window on the screen. Usually (on X) this has the side effect of opening the display.

If the window is already shown then it is restored and raised to the top. This is really convenient because your program can call \mbox{\hyperlink{class_fl___double___window_a4d1c3cbb1184c6785df7465b0c657025}{show()}} at any time, even if the window is already up. It also means that \mbox{\hyperlink{class_fl___double___window_a4d1c3cbb1184c6785df7465b0c657025}{show()}} serves the purpose of raise() in other toolkits.

\mbox{\hyperlink{class_fl___window_a0d75dfcdc8fb797f9247482da1e0152b}{Fl\+\_\+\+Window\+::show(int argc, char $\ast$$\ast$argv)}} is used for top-\/level windows and allows standard arguments to be parsed from the command-\/line.

\begin{DoxyNote}{Note}
For some obscure reasons \mbox{\hyperlink{class_fl___window_a908afc4420fd505e958f6ed394a86acc}{Fl\+\_\+\+Window\+::show()}} resets the current group by calling Fl\+\_\+\+Group\+::current(0). The comments in the code say \char`\"{}get rid of very common user bug\+: forgot end()\char`\"{}. Although this is true it may have unwanted side effects if you \mbox{\hyperlink{class_fl___double___window_a4d1c3cbb1184c6785df7465b0c657025}{show()}} an unrelated window (maybe for an error message or warning) while building a window or any other group widget.
\end{DoxyNote}
