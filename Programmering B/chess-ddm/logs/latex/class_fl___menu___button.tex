\doxysubsection{Detailed Description}
This is a button that when pushed pops up a menu (or hierarchy of menus) defined by an array of \mbox{\hyperlink{struct_fl___menu___item}{Fl\+\_\+\+Menu\+\_\+\+Item}} objects. 

 
\begin{DoxyImage}
\includegraphics[width=5cm]{menu_button.png}
\doxyfigcaption{menu\+\_\+button}
\end{DoxyImage}
 

Normally any mouse button will pop up a menu and it is lined up below the button as shown in the picture. However an \mbox{\hyperlink{class_fl___menu___button}{Fl\+\_\+\+Menu\+\_\+\+Button}} may also control a pop-\/up menu. This is done by setting the \mbox{\hyperlink{class_fl___widget_adda2e8f162b3ea0332eb8d762350bbd5}{type()}}. If \mbox{\hyperlink{class_fl___widget_adda2e8f162b3ea0332eb8d762350bbd5}{type()}} is zero a normal menu button is produced. If it is nonzero then this is a pop-\/up menu. The bits in \mbox{\hyperlink{class_fl___widget_adda2e8f162b3ea0332eb8d762350bbd5}{type()}} indicate what mouse buttons pop up the menu (see \mbox{\hyperlink{class_fl___menu___button_a2c0d1cab5522c531309169acf0e1517e}{Fl\+\_\+\+Menu\+\_\+\+Button\+::popup\+\_\+buttons}}). 

The menu will also pop up in response to shortcuts indicated by putting a \textquotesingle{}\&\textquotesingle{} character in the \mbox{\hyperlink{class_fl___widget_aa327c70689110677d96067da55e227aa}{label()}}. 

Typing the \mbox{\hyperlink{class_fl___menu___aa8080134808c3785579bde1c4ff57f62}{shortcut()}} of any of the menu items will cause callbacks exactly the same as when you pick the item with the mouse. The \textquotesingle{}\&\textquotesingle{} character in menu item names are only looked at when the menu is popped up, however. 

When the user clicks a menu item, \mbox{\hyperlink{class_fl___menu___a760812b12dc473781b132e6a0a7c33e3}{value()}} is set to that item and then\+: \begin{DoxyVerb}- The item's callback is done if one has been set; the
  Fl_Menu_Button is passed as the Fl_Widget* argument, 
  along with any userdata configured for the callback.

- If the item does not have a callback, the Fl_Menu_Button's callback
  is done instead, along with any userdata configured for it.
  The callback can determine which item was picked using
  value(), mvalue(), item_pathname(), etc.
\end{DoxyVerb}
