\doxysubsection{Detailed Description}
A table of widgets or other content.

This is the base class for table widgets.

To be useful it must be subclassed and several virtual functions defined. Normally applications use widgets derived from this widget, and do not use this widget directly; this widget is usually too low level to be used directly by applications.

This widget does {\itshape not} handle the data in the table. The \mbox{\hyperlink{class_fl___table_a3f1f1ccc017d9159c1c98f6f7268bf6d}{draw\+\_\+cell()}} method must be overridden by a subclass to manage drawing the contents of the cells.

This widget can be used in several ways\+:


\begin{DoxyItemize}
\item As a custom widget; see examples/table-\/simple.\+cxx and test/table.\+cxx. Very optimal for even extremely large tables.
\item As a table made up of a single F\+L\+TK widget instanced all over the table, simulating a numeric spreadsheet. See examples/table-\/spreadsheet.\+cxx and examples/table-\/spreadsheet-\/with-\/keyboard-\/nav.\+cxx. Optimal for large tables.
\item As a regular container of F\+L\+TK widgets, one widget per cell. See examples/table-\/as-\/container.\+cxx. {\itshape Not} recommended for large tables.
\end{DoxyItemize}

\begin{DoxyImage}

When acting as part of a custom widget, events on the cells and/or headings generate callbacks when they are clicked by the user. You control when events are generated based on the setting for \mbox{\hyperlink{class_fl___widget_ad4ef380ffb50f4d00fe93f7b6677e9e1}{Fl\+\_\+\+Table\+::when()}}.

When acting as a container for F\+L\+TK widgets, the F\+L\+TK widgets maintain themselves. Although the \mbox{\hyperlink{class_fl___table_a3f1f1ccc017d9159c1c98f6f7268bf6d}{draw\+\_\+cell()}} method must be overridden, its contents can be very simple. See the \mbox{\hyperlink{class_fl___table_a3f1f1ccc017d9159c1c98f6f7268bf6d}{draw\+\_\+cell()}} code in examples/table-\/simple.\+cxx.
