\doxysubsection{Detailed Description}
\mbox{\hyperlink{class_fl___plugin}{Fl\+\_\+\+Plugin}} allows link-\/time and run-\/time integration of binary modules. 

\mbox{\hyperlink{class_fl___plugin}{Fl\+\_\+\+Plugin}} and \mbox{\hyperlink{class_fl___plugin___manager}{Fl\+\_\+\+Plugin\+\_\+\+Manager}} provide a small and simple solution for linking C++ classes at run-\/time, or optionally linking modules at compile time without the need to change the main application.

\mbox{\hyperlink{class_fl___plugin___manager}{Fl\+\_\+\+Plugin\+\_\+\+Manager}} uses static initialisation to create the plugin interface early during startup. Plugins are stored in a temporary database, organized in classes.

Plugins should derive a new class from \mbox{\hyperlink{class_fl___plugin}{Fl\+\_\+\+Plugin}} as a base\+:
