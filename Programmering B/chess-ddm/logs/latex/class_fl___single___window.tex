\doxysubsection{Detailed Description}
This is the same as \mbox{\hyperlink{class_fl___window}{Fl\+\_\+\+Window}}. However, it is possible that some implementations will provide double-\/buffered windows by default. This subclass can be used to force single-\/buffering. This may be useful for modifying existing programs that use incremental update, or for some types of image data, such as a movie flipbook.

{\footnotesize\ttfamily \mbox{\hyperlink{png_8h_aa8c59027f9ab2769342f248709d68d17}{void}} Fl\+\_\+\+Single\+\_\+\+Window\+::show (\begin{DoxyParamCaption}{ }\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [virtual]}}

Puts the window on the screen. Usually (on X) this has the side effect of opening the display.

If the window is already shown then it is restored and raised to the top. This is really convenient because your program can call \mbox{\hyperlink{class_fl___single___window_afe0655393f47614fad9b94490f8711f8}{show()}} at any time, even if the window is already up. It also means that \mbox{\hyperlink{class_fl___single___window_afe0655393f47614fad9b94490f8711f8}{show()}} serves the purpose of raise() in other toolkits.

\mbox{\hyperlink{class_fl___window_a0d75dfcdc8fb797f9247482da1e0152b}{Fl\+\_\+\+Window\+::show(int argc, char $\ast$$\ast$argv)}} is used for top-\/level windows and allows standard arguments to be parsed from the command-\/line.

\begin{DoxyNote}{Note}
For some obscure reasons \mbox{\hyperlink{class_fl___window_a908afc4420fd505e958f6ed394a86acc}{Fl\+\_\+\+Window\+::show()}} resets the current group by calling Fl\+\_\+\+Group\+::current(0). The comments in the code say \char`\"{}get rid of very common user bug\+: forgot end()\char`\"{}. Although this is true it may have unwanted side effects if you \mbox{\hyperlink{class_fl___single___window_afe0655393f47614fad9b94490f8711f8}{show()}} an unrelated window (maybe for an error message or warning) while building a window or any other group widget.
\end{DoxyNote}
