\doxysubsection{Detailed Description}
This struct allows multiple labels to be added to objects that might normally have only one label.

This struct allows a mixed text and/or graphics label to be applied to an object that would normally only have a single (usually text only) label.

Most regular F\+L\+TK widgets now support the ability to associate both images and text with a label but some special cases, notably the non-\/widget \mbox{\hyperlink{struct_fl___menu___item}{Fl\+\_\+\+Menu\+\_\+\+Item}} objects, do not. \mbox{\hyperlink{struct_fl___multi___label}{Fl\+\_\+\+Multi\+\_\+\+Label}} may be used to create menu items that have an icon and text, which would not normally be possible for an \mbox{\hyperlink{struct_fl___menu___item}{Fl\+\_\+\+Menu\+\_\+\+Item}}. For example, \mbox{\hyperlink{struct_fl___multi___label}{Fl\+\_\+\+Multi\+\_\+\+Label}} is used in the New-\/$>$Code submenu in fluid, and others.

Each \mbox{\hyperlink{struct_fl___multi___label}{Fl\+\_\+\+Multi\+\_\+\+Label}} holds two elements, labela and labelb; each may hold either a text label (const char$\ast$) or an image (Fl\+\_\+\+Image$\ast$). When displayed, labela is drawn first and labelb is drawn immediately to its right.

More complex labels might be constructed by setting labelb as another \mbox{\hyperlink{struct_fl___multi___label}{Fl\+\_\+\+Multi\+\_\+\+Label}} and thus chaining up a series of label elements.

When assigning a label element to one of labela or labelb, they should be explicitly cast to (const char$\ast$) if they are not of that type already.
