Copies the data pointed to by {\ttfamily stuff} to the selection buffer ({\ttfamily destination} is 0), the clipboard ({\ttfamily destination} is 1), or both ({\ttfamily destination} is 2). Copying to both is only relevant on X11, on other platforms it maps to the clipboard (1). {\ttfamily len} is the number of relevant bytes in {\ttfamily stuff}. {\ttfamily type} is always \mbox{\hyperlink{group__fl__clipboard_gaf42b5221c0e00765ffcfc81cfce3c629}{Fl\+::clipboard\+\_\+plain\+\_\+text}}. The selection buffer is used for middle-\/mouse pastes and for drag-\/and-\/drop selections. The clipboard is used for traditional copy/cut/paste operations.

\begin{DoxyNote}{Note}
This function is, at present, intended only to copy U\+T\+F-\/8 encoded textual data. To copy graphical data, use the \mbox{\hyperlink{class_fl___copy___surface}{Fl\+\_\+\+Copy\+\_\+\+Surface}} class. The {\ttfamily type} argument may allow in the future to copy other kinds of data. 
\end{DoxyNote}

{\footnotesize\ttfamily static \mbox{\hyperlink{png_8h_aa8c59027f9ab2769342f248709d68d17}{void}} Fl\+::paste (\begin{DoxyParamCaption}\item[{\mbox{\hyperlink{class_fl___widget}{Fl\+\_\+\+Widget}} \&}]{receiver,  }\item[{int}]{source,  }\item[{\mbox{\hyperlink{zconf_8h_a2c212835823e3c54a8ab6d95c652660e}{const}} char $\ast$}]{type = {\ttfamily \mbox{\hyperlink{group__fl__clipboard_gaf42b5221c0e00765ffcfc81cfce3c629}{Fl\+::clipboard\+\_\+plain\+\_\+text}}} }\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [static]}}

Pastes the data from the selection buffer ({\ttfamily source} is 0) or the clipboard ({\ttfamily source} is 1) into {\ttfamily receiver}. ~\newline


The selection buffer ({\ttfamily source} is 0) is used for middle-\/mouse pastes and for drag-\/and-\/drop selections. The clipboard ({\ttfamily source} is 1) is used for copy/cut/paste operations.

If {\ttfamily source} is 1, the optional {\ttfamily type} argument indicates what type of data is requested from the clipboard. At present, \mbox{\hyperlink{group__fl__clipboard_gaf42b5221c0e00765ffcfc81cfce3c629}{Fl\+::clipboard\+\_\+plain\+\_\+text}} (requesting text data) and \mbox{\hyperlink{group__fl__clipboard_gac5d1983ba6ee5e88bd3b157f76449d7b}{Fl\+::clipboard\+\_\+image}} (requesting image data) are possible. Set things up so the handle function of the {\ttfamily receiver} widget will be called with an F\+L\+\_\+\+P\+A\+S\+TE event some time in the future if the clipboard does contain data of the requested type. While processing the F\+L\+\_\+\+P\+A\+S\+TE event\+: \begin{DoxyItemize}
\item if {\ttfamily type} is \mbox{\hyperlink{group__fl__clipboard_gaf42b5221c0e00765ffcfc81cfce3c629}{Fl\+::clipboard\+\_\+plain\+\_\+text}}, the text string from the specified {\ttfamily source} is in \mbox{\hyperlink{group__fl__events_ga6647c55948fe1d8be9367267529e9c54}{Fl\+::event\+\_\+text()}} with U\+T\+F-\/8 encoding, and the number of bytes in \mbox{\hyperlink{group__fl__events_ga38f2de89fbdf59ad2cd4dca93f472911}{Fl\+::event\+\_\+length()}}. If \mbox{\hyperlink{group__fl__clipboard_ga0f30d10ec79056fbdbf8d2f4dca4d565}{Fl\+::paste()}} gets called during the drop step of a files-\/drag-\/and-\/drop operation, \mbox{\hyperlink{group__fl__events_ga6647c55948fe1d8be9367267529e9c54}{Fl\+::event\+\_\+text()}} contains a list of filenames (see events\+\_\+dnd). \item if {\ttfamily type} is \mbox{\hyperlink{group__fl__clipboard_gac5d1983ba6ee5e88bd3b157f76449d7b}{Fl\+::clipboard\+\_\+image}}, the pointer returned by \mbox{\hyperlink{group__fl__events_ga5964867d1d60792c45bb28082c257624}{Fl\+::event\+\_\+clipboard()}} can be safely cast to type \mbox{\hyperlink{class_fl___image}{Fl\+\_\+\+Image}} $\ast$ to obtain a pointer to the pasted image. Furthermore, starting with F\+L\+TK 1.\+3.\+4, the image is of type \mbox{\hyperlink{class_fl___r_g_b___image}{Fl\+\_\+\+R\+G\+B\+\_\+\+Image}} across all platforms. If {\ttfamily receiver} accepts the clipboard image, receiver.\+handle() should return 1 and the application should take ownership of this image (that is, delete it after use). Conversely, if receiver.\+handle() returns 0, the application must not use the image.\end{DoxyItemize}
The receiver should be prepared to be called {\itshape directly} by this, or for it to happen {\itshape later}, or possibly {\itshape not at all}. This allows the window system to take as long as necessary to retrieve the paste buffer (or even to screw up completely) without complex and error-\/prone synchronization code in F\+L\+TK.

\begin{DoxyParagraph}{Platform details for image data\+:}
\begin{DoxyItemize}
\item Unix/\+Linux platform\+: Clipboard images in P\+NG or B\+MP formats are recognized. Requires linking with the fltk\+\_\+images library. \item M\+S\+Windows platform\+: Both bitmap and vectorial (Enhanced metafile) data from clipboard can be pasted as image data. \item Mac OS X platform\+: Both bitmap (T\+I\+FF) and vectorial (P\+DF) data from clipboard can be pasted as image data. \end{DoxyItemize}

\end{DoxyParagraph}
