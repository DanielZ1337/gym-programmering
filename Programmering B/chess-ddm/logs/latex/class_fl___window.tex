\doxysubsection{Detailed Description}
This widget produces an actual window. This can either be a main window, with a border and title and all the window management controls, or a \char`\"{}subwindow\char`\"{} inside a window. This is controlled by whether or not the window has a \mbox{\hyperlink{class_fl___widget_a6f55261683ecaf68d70e424213d030b0}{parent()}}.

Once you create a window, you usually add children \mbox{\hyperlink{class_fl___widget}{Fl\+\_\+\+Widget}} \textquotesingle{}s to it by using window-\/$>$add(child) for each new widget. See \mbox{\hyperlink{class_fl___group}{Fl\+\_\+\+Group}} for more information on how to add and remove children.

There are several subclasses of \mbox{\hyperlink{class_fl___window}{Fl\+\_\+\+Window}} that provide double-\/buffering, overlay, menu, and Open\+GL support.

The window\textquotesingle{}s callback is done if the user tries to close a window using the window manager and \mbox{\hyperlink{group__fl__windows_gaf0938156f04e1babebaa8eb75a8d0fce}{Fl\+::modal()}} is zero or equal to the window. \mbox{\hyperlink{class_fl___window}{Fl\+\_\+\+Window}} has a default callback that calls \mbox{\hyperlink{class_fl___window_a42fbeddb68a203ff9123d33156a29fc7}{Fl\+\_\+\+Window\+::hide()}}.

{\footnotesize\ttfamily \mbox{\hyperlink{png_8h_aa8c59027f9ab2769342f248709d68d17}{void}} Fl\+\_\+\+Window\+::clear\+\_\+modal\+\_\+states (\begin{DoxyParamCaption}{ }\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [inline]}}

Clears the \char`\"{}modal\char`\"{} flags and converts a \char`\"{}modal\char`\"{} or \char`\"{}non-\/modal\char`\"{} window back into a \char`\"{}normal\char`\"{} window.

Note that there are {\itshape three} states for a window\+: modal, non-\/modal, and normal.

You can not change the \char`\"{}modality\char`\"{} of a window whilst it is shown, so it is necessary to first \mbox{\hyperlink{class_fl___window_a42fbeddb68a203ff9123d33156a29fc7}{hide()}} the window, change its \char`\"{}modality\char`\"{} as required, then re-\/show the window for the new state to take effect.

This method can also be used to change a \char`\"{}modal\char`\"{} window into a \char`\"{}non-\/modal\char`\"{} one. On several supported platforms, the \char`\"{}modal\char`\"{} state over-\/rides the \char`\"{}non-\/modal\char`\"{} state, so the \char`\"{}modal\char`\"{} state must be cleared before the window can be set into the \char`\"{}non-\/modal\char`\"{} state. In general, the following sequence should work\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{win-\/>hide();}
\DoxyCodeLine{win-\/>clear\_modal\_states();}
\DoxyCodeLine{\textcolor{comment}{//Set win to new state as desired, or leave "normal", e.g...}}
\DoxyCodeLine{win-\/>set\_non\_modal();}
\DoxyCodeLine{win-\/>show();}
\end{DoxyCode}


\begin{DoxyNote}{Note}
Under some window managers, the sequence of hiding the window and changing its modality will often cause it to be re-\/displayed at a different position when it is subsequently shown. This is an irritating feature but appears to be unavoidable at present. As a result we would advise to use this method only when absolutely necessary.
\end{DoxyNote}

Changes the size and position of the window. If \mbox{\hyperlink{class_fl___window_aa09300542b2d8b49fc69a037bdd3b742}{shown()}} is true, these changes are communicated to the window server (which may refuse that size and cause a further resize). If \mbox{\hyperlink{class_fl___window_aa09300542b2d8b49fc69a037bdd3b742}{shown()}} is false, the size and position are used when \mbox{\hyperlink{class_fl___window_a908afc4420fd505e958f6ed394a86acc}{show()}} is called. You can also call the \mbox{\hyperlink{class_fl___widget}{Fl\+\_\+\+Widget}} methods size(x,y) and position(w,h), which are inline wrappers for this virtual function.

A top-\/level window can not force, but merely suggest a position and size to the operating system. The window manager may not be willing or able to display a window at the desired position or with the given dimensions. It is up to the application developer to verify window parameters after the resize request.

Puts the window on the screen. Usually (on X) this has the side effect of opening the display.

If the window is already shown then it is restored and raised to the top. This is really convenient because your program can call \mbox{\hyperlink{class_fl___window_a908afc4420fd505e958f6ed394a86acc}{show()}} at any time, even if the window is already up. It also means that \mbox{\hyperlink{class_fl___window_a908afc4420fd505e958f6ed394a86acc}{show()}} serves the purpose of raise() in other toolkits.

\mbox{\hyperlink{class_fl___window_a0d75dfcdc8fb797f9247482da1e0152b}{Fl\+\_\+\+Window\+::show(int argc, char $\ast$$\ast$argv)}} is used for top-\/level windows and allows standard arguments to be parsed from the command-\/line.

\begin{DoxyNote}{Note}
For some obscure reasons \mbox{\hyperlink{class_fl___window_a908afc4420fd505e958f6ed394a86acc}{Fl\+\_\+\+Window\+::show()}} resets the current group by calling Fl\+\_\+\+Group\+::current(0). The comments in the code say \char`\"{}get rid of very common user bug\+: forgot end()\char`\"{}. Although this is true it may have unwanted side effects if you \mbox{\hyperlink{class_fl___window_a908afc4420fd505e958f6ed394a86acc}{show()}} an unrelated window (maybe for an error message or warning) while building a window or any other group widget.
\end{DoxyNote}

{\footnotesize\ttfamily \mbox{\hyperlink{png_8h_aa8c59027f9ab2769342f248709d68d17}{void}} Fl\+\_\+\+Window\+::size\+\_\+range (\begin{DoxyParamCaption}\item[{int}]{minw,  }\item[{int}]{minh,  }\item[{int}]{maxw = {\ttfamily 0},  }\item[{int}]{maxh = {\ttfamily 0},  }\item[{int}]{dw = {\ttfamily 0},  }\item[{int}]{dh = {\ttfamily 0},  }\item[{int}]{aspect = {\ttfamily 0} }\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [inline]}}

Sets the allowable range the user can resize this window to. This only works for top-\/level windows. 
\begin{DoxyItemize}
\item {\ttfamily minw} and {\ttfamily minh} are the smallest the window can be. Either value must be greater than 0. 
\item {\ttfamily maxw} and {\ttfamily maxh} are the largest the window can be. If either is {\itshape equal} to the minimum then you cannot resize in that direction. If either is zero then F\+L\+TK picks a maximum size in that direction such that the window will fill the screen. 
\item {\ttfamily dw} and {\ttfamily dh} are size increments. The window will be constrained to widths of minw + N $\ast$ dw, where N is any non-\/negative integer. If these are less or equal to 1 they are ignored (this is ignored on W\+I\+N32). 
\item {\ttfamily aspect} is a flag that indicates that the window should preserve its aspect ratio. This only works if both the maximum and minimum have the same aspect ratio (ignored on W\+I\+N32 and by many X window managers).  
\end{DoxyItemize}

If this function is not called, F\+L\+TK tries to figure out the range from the setting of \mbox{\hyperlink{class_fl___group_ab0ed0d1974d7185594687849f386c5f3}{resizable()}}\+: 
\begin{DoxyItemize}
\item If \mbox{\hyperlink{class_fl___group_ab0ed0d1974d7185594687849f386c5f3}{resizable()}} is N\+U\+LL (this is the default) then the window cannot be resized and the resize border and max-\/size control will not be displayed for the window. 
\item If either dimension of \mbox{\hyperlink{class_fl___group_ab0ed0d1974d7185594687849f386c5f3}{resizable()}} is less than 100, then that is considered the minimum size. Otherwise the \mbox{\hyperlink{class_fl___group_ab0ed0d1974d7185594687849f386c5f3}{resizable()}} has a minimum size of 100. 
\item If either dimension of \mbox{\hyperlink{class_fl___group_ab0ed0d1974d7185594687849f386c5f3}{resizable()}} is zero, then that is also the maximum size (so the window cannot resize in that direction). 
\end{DoxyItemize}

It is undefined what happens if the current size does not fit in the constraints passed to \mbox{\hyperlink{class_fl___window_aa0f717a52f23e54d7db8fee71a5f299e}{size\+\_\+range()}}. \mbox{\Hypertarget{class_fl___window_af947957fd93b91b528e72b126201d534}\label{class_fl___window_af947957fd93b91b528e72b126201d534}}
