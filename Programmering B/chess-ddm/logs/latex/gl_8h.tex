\doxysubsection{Detailed Description}
This file defines wrapper functions for Open\+GL in F\+L\+TK

To use Open\+GL from within an F\+L\+TK application you M\+U\+ST use gl\+\_\+visual() to select the default visual before doing show() on any windows. Mesa will crash if yoy try to use a visual not returned by glx\+Choose\+Vidual.

This does not work with \mbox{\hyperlink{class_fl___double___window}{Fl\+\_\+\+Double\+\_\+\+Window}}\textquotesingle{}s! It will try to draw into the front buffer. Depending on the system this will either crash or do nothing (when pixmaps are being used as back buffer and GL is being done by hardware), work correctly (when GL is done with software, such as Mesa), or draw into the front buffer and be erased when the buffers are swapped (when double buffer hardware is being used)
