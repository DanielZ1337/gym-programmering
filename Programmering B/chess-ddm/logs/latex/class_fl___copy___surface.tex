\doxysubsection{Detailed Description}
Supports copying of graphical data to the clipboard.

~\newline
 After creation of an \mbox{\hyperlink{class_fl___copy___surface}{Fl\+\_\+\+Copy\+\_\+\+Surface}} object, call \mbox{\hyperlink{class_fl___copy___surface_a303a6d9cd0f8df6d7e24589e6c88e54c}{set\+\_\+current()}} on it, and all subsequent graphics requests will be recorded in the clipboard. It\textquotesingle{}s possible to draw widgets (using \mbox{\hyperlink{class_fl___copy___surface_a41846b9b963343d618376c80e6aa32dc}{Fl\+\_\+\+Copy\+\_\+\+Surface\+::draw()}} ) or to use any of the \mbox{\hyperlink{group__fl__drawings}{Drawing functions}} or the \mbox{\hyperlink{group__fl__attributes}{Color \& Font functions}}. Finally, delete the \mbox{\hyperlink{class_fl___copy___surface}{Fl\+\_\+\+Copy\+\_\+\+Surface}} object to load the clipboard with the graphical data. ~\newline
 Fl\+\_\+\+G\+L\+\_\+\+Window \textquotesingle{}s can be copied to the clipboard as well. ~\newline

 Platform details\+: \begin{DoxyItemize}
\item M\+S\+Windows\+: Transparent R\+GB images copy without transparency. The graphical data are copied to the clipboard as an \textquotesingle{}enhanced metafile\textquotesingle{}. \item Mac OS\+: The graphical data are copied to the clipboard (a.\+k.\+a. pasteboard) in two \textquotesingle{}flavors\textquotesingle{}\+: 1) in vectorial form as P\+DF data; 2) in bitmap form as a T\+I\+FF image. Applications to which the clipboard content is pasted can use the flavor that suits them best. \item X11\+: the graphical data are copied to the clipboard as an image in B\+MP format. \end{DoxyItemize}
