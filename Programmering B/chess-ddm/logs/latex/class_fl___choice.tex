\doxysubsection{Detailed Description}
A button that is used to pop up a menu. 

This is a button that, when pushed, pops up a menu (or hierarchy of menus) defined by an array of \mbox{\hyperlink{struct_fl___menu___item}{Fl\+\_\+\+Menu\+\_\+\+Item}} objects. Motif calls this an Option\+Button.

The only difference between this and a \mbox{\hyperlink{class_fl___menu___button}{Fl\+\_\+\+Menu\+\_\+\+Button}} is that the name of the most recent chosen menu item is displayed inside the box, while the label is displayed outside the box. However, since the use of this is most often to control a single variable rather than do individual callbacks, some of the \mbox{\hyperlink{class_fl___menu___button}{Fl\+\_\+\+Menu\+\_\+\+Button}} methods are redescribed here in those terms.

When the user clicks a menu item, \mbox{\hyperlink{class_fl___choice_a9da9b4ce82f756bd006ca5af6aaff048}{value()}} is set to that item and then\+: \begin{DoxyVerb}- The item's callback is done if one has been set; the
  Fl_Choice is passed as the Fl_Widget* argument, 
  along with any userdata configured for the callback.

- If the item does not have a callback, the Fl_Choice widget's
  callback is done instead, along with any userdata configured
  for it.  The callback can determine which item was picked using
  value(), mvalue(), item_pathname(), etc.
\end{DoxyVerb}


All three mouse buttons pop up the menu. The Forms behavior of the first two buttons to increment/decrement the choice is not implemented. This could be added with a subclass, however.

The menu will also pop up in response to shortcuts indicated by putting a \textquotesingle{}\&\textquotesingle{} character in the \mbox{\hyperlink{class_fl___widget_aa327c70689110677d96067da55e227aa}{label()}}. See \mbox{\hyperlink{class_fl___button_ad0c694762b456cbb3c42927b2129fdd4}{Fl\+\_\+\+Button\+::shortcut(int s)}} for a description of this.

Typing the \mbox{\hyperlink{class_fl___menu___aa8080134808c3785579bde1c4ff57f62}{shortcut()}} of any of the items will do exactly the same as when you pick the item with the mouse. The \textquotesingle{}\&\textquotesingle{} character in item names are only looked at when the menu is popped up, however.

Handles the specified event. You normally don\textquotesingle{}t call this method directly, but instead let F\+L\+TK do it when the user interacts with the widget.

When implemented in a widget, this function must return 0 if the widget does not use the event or 1 otherwise.

Most of the time, you want to call the inherited \mbox{\hyperlink{class_fl___choice_ad7b67a4db58f558401b0593e13c6ca34}{handle()}} method in your overridden method so that you don\textquotesingle{}t short-\/circuit events that you don\textquotesingle{}t handle.
