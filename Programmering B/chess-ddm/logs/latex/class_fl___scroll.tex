\doxysubsection{Detailed Description}
This container widget lets you maneuver around a set of widgets much larger than your window. If the child widgets are larger than the size of this object then scrollbars will appear so that you can scroll over to them\+:  
\begin{DoxyImage}
\includegraphics[width=4cm]{Fl_Scroll.png}
\doxyfigcaption{Fl\+\_\+\+Scroll}
\end{DoxyImage}


If all of the child widgets are packed together into a solid rectangle then you want to set \mbox{\hyperlink{class_fl___widget_a9d1f48fbbe4d885e81df38b2a93b614f}{box()}} to F\+L\+\_\+\+N\+O\+\_\+\+B\+OX or one of the \+\_\+\+F\+R\+A\+ME types. This will result in the best output. However, if the child widgets are a sparse arrangement you must set \mbox{\hyperlink{class_fl___widget_a9d1f48fbbe4d885e81df38b2a93b614f}{box()}} to a real \+\_\+\+B\+OX type. This can result in some blinking during redrawing, but that can be solved by using a \mbox{\hyperlink{class_fl___double___window}{Fl\+\_\+\+Double\+\_\+\+Window}}.

By default you can scroll in both directions, and the scrollbars disappear if the data will fit in the area of the scroll.

Use \mbox{\hyperlink{class_fl___widget_adda2e8f162b3ea0332eb8d762350bbd5}{Fl\+\_\+\+Scroll\+::type()}} to change this as follows \+:


\begin{DoxyItemize}
\item 0 -\/ No scrollbars
\item \mbox{\hyperlink{class_fl___scroll_abd3162f481625843f3f762db73f902dbaac33d5172b559df56ef253987a660e72}{Fl\+\_\+\+Scroll\+::\+H\+O\+R\+I\+Z\+O\+N\+T\+AL}} -\/ Only a horizontal scrollbar.
\item \mbox{\hyperlink{class_fl___scroll_abd3162f481625843f3f762db73f902dba4b873bab7be947f316ba3460584253fd}{Fl\+\_\+\+Scroll\+::\+V\+E\+R\+T\+I\+C\+AL}} -\/ Only a vertical scrollbar.
\item \mbox{\hyperlink{class_fl___scroll_abd3162f481625843f3f762db73f902dbaa86133f933846b5579f8cef4ddae10a6}{Fl\+\_\+\+Scroll\+::\+B\+O\+TH}} -\/ The default is both scrollbars.
\item \mbox{\hyperlink{class_fl___scroll_abd3162f481625843f3f762db73f902dba5e529b94b7f4897f1ccbe9f1be66d7b2}{Fl\+\_\+\+Scroll\+::\+H\+O\+R\+I\+Z\+O\+N\+T\+A\+L\+\_\+\+A\+L\+W\+A\+YS}} -\/ Horizontal scrollbar always on, vertical always off.
\item \mbox{\hyperlink{class_fl___scroll_abd3162f481625843f3f762db73f902dbab5cd556262cd4b1a1541ab3d9dea35a6}{Fl\+\_\+\+Scroll\+::\+V\+E\+R\+T\+I\+C\+A\+L\+\_\+\+A\+L\+W\+A\+YS}} -\/ Vertical scrollbar always on, horizontal always off.
\item \mbox{\hyperlink{class_fl___scroll_abd3162f481625843f3f762db73f902dba03f3bca7b9640e2a8f0e1ff31fe6d4f8}{Fl\+\_\+\+Scroll\+::\+B\+O\+T\+H\+\_\+\+A\+L\+W\+A\+YS}} -\/ Both always on.
\end{DoxyItemize}

Use {\bfseries{ scrollbar.\+align(int) ( see void \mbox{\hyperlink{class_fl___widget_a7dcd6fc8c6108fb68a00bf922cc79022}{Fl\+\_\+\+Widget\+::align(\+Fl\+\_\+\+Align)}} ) \+:}} to change what side the scrollbars are drawn on.

If the F\+L\+\_\+\+A\+L\+I\+G\+N\+\_\+\+L\+E\+FT bit is on, the vertical scrollbar is on the left. If the F\+L\+\_\+\+A\+L\+I\+G\+N\+\_\+\+T\+OP bit is on, the horizontal scrollbar is on the top. Note that only the alignment flags in scrollbar are considered. The flags in hscrollbar however are ignored.

This widget can also be used to pan around a single child widget \char`\"{}canvas\char`\"{}. This child widget should be of your own class, with a \mbox{\hyperlink{class_fl___scroll_a3f0651126d937ff5e87a115c96b01e47}{draw()}} method that draws the contents. The scrolling is done by changing the \mbox{\hyperlink{class_fl___widget_abea0b9ad493d58bf7fda8d3c81d477b7}{x()}} and \mbox{\hyperlink{class_fl___widget_aec1aded9efdf5f23dfac8cc725cefc4a}{y()}} of the widget, so this child must use the \mbox{\hyperlink{class_fl___widget_abea0b9ad493d58bf7fda8d3c81d477b7}{x()}} and \mbox{\hyperlink{class_fl___widget_aec1aded9efdf5f23dfac8cc725cefc4a}{y()}} to position its drawing. To speed up drawing it should test \mbox{\hyperlink{group__fl__drawings_ga2bd899d466196ffc26cc61542ee10539}{fl\+\_\+not\+\_\+clipped(int x,int y,int w,int h)}} to find out if a particular area of the widget must be drawn.

Another very useful child is a single \mbox{\hyperlink{class_fl___pack}{Fl\+\_\+\+Pack}}, which is itself a group that packs its children together and changes size to surround them. Filling the \mbox{\hyperlink{class_fl___pack}{Fl\+\_\+\+Pack}} with \mbox{\hyperlink{class_fl___tabs}{Fl\+\_\+\+Tabs}} groups (and then putting normal widgets inside those) gives you a very powerful scrolling list of individually-\/openable panels.

Fluid lets you create these, but you can only lay out objects that fit inside the \mbox{\hyperlink{class_fl___scroll}{Fl\+\_\+\+Scroll}} without scrolling. Be sure to leave space for the scrollbars, as Fluid won\textquotesingle{}t show these either.

{\itshape You cannot use \mbox{\hyperlink{class_fl___window}{Fl\+\_\+\+Window}} as a child of this since the clipping is not conveyed to it when drawn, and it will draw over the scrollbars and neighboring objects.}

Changes the size or position of the widget.

This is a virtual function so that the widget may implement its own handling of resizing. The default version does {\itshape not} call the \mbox{\hyperlink{class_fl___widget_aa63ce68cbf4620cf8750b868368ea02b}{redraw()}} method, but instead relies on the parent widget to do so because the parent may know a faster way to update the display, such as scrolling from the old position. ~\newline


Some window managers under X11 call \mbox{\hyperlink{class_fl___scroll_a4328309a02ce7af89f569ef3e0314d80}{resize()}} a lot more often than needed. Please verify that the position or size of a widget did actually change before doing any extensive calculations.

position(\+X, Y) is a shortcut for resize(X, Y, \mbox{\hyperlink{class_fl___widget_ad59ece778aaf15acc4dd01a8bf5f8012}{w()}}, \mbox{\hyperlink{class_fl___widget_aeeabd61e2d10c76cd6600dbbfe951f71}{h()}}), and size(\+W, H) is a shortcut for resize(\mbox{\hyperlink{class_fl___widget_abea0b9ad493d58bf7fda8d3c81d477b7}{x()}}, \mbox{\hyperlink{class_fl___widget_aec1aded9efdf5f23dfac8cc725cefc4a}{y()}}, W, H).

Normally you should not need this method, and should use \mbox{\hyperlink{class_fl_aca2f98b26f2552b82dd245b8094a41aa}{Fl\+::scrollbar\+\_\+size(int)}} instead to manage the size of A\+LL your widgets\textquotesingle{} scrollbars. Only use T\+H\+IS method if you really need to override the global scrollbar size. The need for this should be rare.

Setting {\ttfamily new\+Size} to the special value of 0 causes the widget to track the global \mbox{\hyperlink{class_fl_ae861b8bd4f2d4dcbec7be697e8b4b80e}{Fl\+::scrollbar\+\_\+size()}}, which is the default.
