\doxysubsection{Detailed Description}
Tree widget item. 

This class is a single tree item, and manages all of the item\textquotesingle{}s attributes. \mbox{\hyperlink{class_fl___tree___item}{Fl\+\_\+\+Tree\+\_\+\+Item}} is used by \mbox{\hyperlink{class_fl___tree}{Fl\+\_\+\+Tree}}, which is comprised of many instances of \mbox{\hyperlink{class_fl___tree___item}{Fl\+\_\+\+Tree\+\_\+\+Item}}.

\mbox{\hyperlink{class_fl___tree___item}{Fl\+\_\+\+Tree\+\_\+\+Item}} is hierarchical; it dynamically manages an \mbox{\hyperlink{class_fl___tree___item___array}{Fl\+\_\+\+Tree\+\_\+\+Item\+\_\+\+Array}} of children that are themselves instances of \mbox{\hyperlink{class_fl___tree___item}{Fl\+\_\+\+Tree\+\_\+\+Item}}. Each item can have zero or more children. When an item has children, \mbox{\hyperlink{class_fl___tree___item_a9d045540a28f635706eb8dfc8d604832}{close()}} and \mbox{\hyperlink{class_fl___tree___item_a560d5f506413ece45df791d474d3c520}{open()}} can be used to hide or show them.

Items have their own attributes; font size, face, color. Items maintain their own hierarchy of children.

When you make changes to items, you\textquotesingle{}ll need to tell the tree to redraw() for the changes to show up.

New 1.\+3.\+3 A\+BI feature\+: You can define custom items by either adding a custom widget to the item with \mbox{\hyperlink{class_fl___tree___item_ad6f913414d8af47f16289b6c0712612a}{Fl\+\_\+\+Tree\+\_\+\+Item\+::widget()}}, or override the draw\+\_\+item\+\_\+content() method if you want to just redefine how the label is drawn.
