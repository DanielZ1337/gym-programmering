\doxysubsection{Detailed Description}
The \mbox{\hyperlink{class_fl___menu___window}{Fl\+\_\+\+Menu\+\_\+\+Window}} widget is a window type used for menus. By default the window is drawn in the hardware overlay planes if they are available so that the menu don\textquotesingle{}t force the rest of the window to redraw.

{\footnotesize\ttfamily \mbox{\hyperlink{png_8h_aa8c59027f9ab2769342f248709d68d17}{void}} Fl\+\_\+\+Menu\+\_\+\+Window\+::show (\begin{DoxyParamCaption}{ }\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [virtual]}}

Puts the window on the screen. Usually (on X) this has the side effect of opening the display.

If the window is already shown then it is restored and raised to the top. This is really convenient because your program can call \mbox{\hyperlink{class_fl___menu___window_a5658ae8d0587eeb640c44da3b2587441}{show()}} at any time, even if the window is already up. It also means that \mbox{\hyperlink{class_fl___menu___window_a5658ae8d0587eeb640c44da3b2587441}{show()}} serves the purpose of raise() in other toolkits.

\mbox{\hyperlink{class_fl___window_a0d75dfcdc8fb797f9247482da1e0152b}{Fl\+\_\+\+Window\+::show(int argc, char $\ast$$\ast$argv)}} is used for top-\/level windows and allows standard arguments to be parsed from the command-\/line.

\begin{DoxyNote}{Note}
For some obscure reasons \mbox{\hyperlink{class_fl___window_a908afc4420fd505e958f6ed394a86acc}{Fl\+\_\+\+Window\+::show()}} resets the current group by calling Fl\+\_\+\+Group\+::current(0). The comments in the code say \char`\"{}get rid of very common user bug\+: forgot end()\char`\"{}. Although this is true it may have unwanted side effects if you \mbox{\hyperlink{class_fl___menu___window_a5658ae8d0587eeb640c44da3b2587441}{show()}} an unrelated window (maybe for an error message or warning) while building a window or any other group widget.
\end{DoxyNote}
