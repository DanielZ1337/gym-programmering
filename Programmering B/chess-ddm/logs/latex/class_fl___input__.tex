\doxysubsection{Detailed Description}
This class provides a low-\/overhead text input field.

This is a virtual base class below \mbox{\hyperlink{class_fl___input}{Fl\+\_\+\+Input}}. It has all the same interfaces, but lacks the \mbox{\hyperlink{class_fl___widget_a3521aba25eda761620953dd49d335ea7}{handle()}} and \mbox{\hyperlink{class_fl___widget_a1acb38c6b3cb40452ad02ccfeedbac8a}{draw()}} method. You may want to subclass it if you are one of those people who likes to change how the editing keys work. It may also be useful for adding scrollbars to the input field.

This can act like any of the subclasses of \mbox{\hyperlink{class_fl___input}{Fl\+\_\+\+Input}}, by setting \mbox{\hyperlink{class_fl___widget_adda2e8f162b3ea0332eb8d762350bbd5}{type()}} to one of the following values\+:


\begin{DoxyCode}{0}

\DoxyCodeLine{\textcolor{preprocessor}{\#define FL\_FLOAT\_INPUT     1}}
\DoxyCodeLine{\textcolor{preprocessor}{\#define FL\_INT\_INPUT           2}}
\DoxyCodeLine{\textcolor{preprocessor}{\#define FL\_MULTILINE\_INPUT     4}}
\DoxyCodeLine{\textcolor{preprocessor}{\#define FL\_SECRET\_INPUT    5}}
\DoxyCodeLine{\textcolor{preprocessor}{\#define FL\_INPUT\_TYPE          7}}
\DoxyCodeLine{\textcolor{preprocessor}{\#define FL\_INPUT\_READONLY      8}}
\DoxyCodeLine{\textcolor{preprocessor}{\#define FL\_NORMAL\_OUTPUT       (FL\_NORMAL\_INPUT | FL\_INPUT\_READONLY)}}
\DoxyCodeLine{\textcolor{preprocessor}{\#define FL\_MULTILINE\_OUTPUT    (FL\_MULTILINE\_INPUT | FL\_INPUT\_READONLY)}}
\DoxyCodeLine{\textcolor{preprocessor}{\#define FL\_INPUT\_WRAP          16}}
\DoxyCodeLine{\textcolor{preprocessor}{\#define FL\_MULTILINE\_INPUT\_WRAP  (FL\_MULTILINE\_INPUT | FL\_INPUT\_WRAP)}}
\DoxyCodeLine{\textcolor{preprocessor}{\#define FL\_MULTILINE\_OUTPUT\_WRAP (FL\_MULTILINE\_INPUT | FL\_INPUT\_READONLY | FL\_INPUT\_WRAP)}}
\end{DoxyCode}


All variables that represent an index into a text buffer are byte-\/oriented, not character oriented, counting from 0 (at or before the first character) to \mbox{\hyperlink{class_fl___input___aec34d9c5cd81a73f0fe47507afa2a7dc}{size()}} (at the end of the buffer, after the last byte). Since U\+T\+F-\/8 characters can be up to six bytes long, simply incrementing such an index will not reliably advance to the next character in the text buffer.

Indices and pointers into the text buffer should always point at a 7 bit A\+S\+C\+II character or the beginning of a U\+T\+F-\/8 character sequence. Behavior for false U\+T\+F-\/8 sequences and pointers into the middle of a sequence are undefined.

The default version does {\itshape not} call the \mbox{\hyperlink{class_fl___widget_aa63ce68cbf4620cf8750b868368ea02b}{redraw()}} method, but instead relies on the parent widget to do so because the parent may know a faster way to update the display, such as scrolling from the old position. ~\newline


Some window managers under X11 call \mbox{\hyperlink{class_fl___input___a1b4c33d2caa558b8f028c3cad2b433c2}{resize()}} a lot more often than needed. Please verify that the position or size of a widget did actually change before doing any extensive calculations.

position(\+X, Y) is a shortcut for resize(X, Y, \mbox{\hyperlink{class_fl___widget_ad59ece778aaf15acc4dd01a8bf5f8012}{w()}}, \mbox{\hyperlink{class_fl___widget_aeeabd61e2d10c76cd6600dbbfe951f71}{h()}}), and size(\+W, H) is a shortcut for resize(\mbox{\hyperlink{class_fl___widget_abea0b9ad493d58bf7fda8d3c81d477b7}{x()}}, \mbox{\hyperlink{class_fl___widget_aec1aded9efdf5f23dfac8cc725cefc4a}{y()}}, W, H).

Sets whether the Tab key does focus navigation, or inserts tab characters into \mbox{\hyperlink{class_fl___multiline___input}{Fl\+\_\+\+Multiline\+\_\+\+Input}}.

By default this flag is enabled to provide the \textquotesingle{}normal\textquotesingle{} behavior most users expect; Tab navigates focus to the next widget. To inserting an actual Tab character, users can use Ctrl-\/I or copy/paste.

Disabling this flag gives the old F\+L\+TK behavior where Tab inserts a tab character into the text field, in which case only the mouse can be used to navigate to the next field.

History\+: This flag was provided for backwards support of F\+L\+TK\textquotesingle{}s old 1.\+1.\+x behavior where Tab inserts a tab character instead of navigating focus to the next widget. This behavior was unique to \mbox{\hyperlink{class_fl___multiline___input}{Fl\+\_\+\+Multiline\+\_\+\+Input}}. With the advent of \mbox{\hyperlink{class_fl___text___editor}{Fl\+\_\+\+Text\+\_\+\+Editor}}, this old behavior has been deprecated.
